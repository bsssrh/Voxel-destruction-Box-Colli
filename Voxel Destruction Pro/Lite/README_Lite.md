# Voxel Destruction Pro/Lite — быстрый старт

Этот файл описывает минимальную настройку Lite‑компонентов для загрузки `.vox` и разрушений.

## 1. Добавление `LiteVoxelManager` в сцену

1. Создайте пустой GameObject в сцене (например, `LiteVoxelManager`).
2. Добавьте компонент **LiteVoxelManager**.
3. В инспекторе задайте:
   - **Default Material** — материал для меша вокселей.
   - **Mesh Settings** — `MeshSettingsObj` (можно оставить пустым, будет создано по умолчанию).
   - **Dynamic Settings** — `DynSettings` (если пусто, используется режим `Remove` по умолчанию).

`LiteVoxelManager` должен существовать в сцене до загрузки `.vox` (его ищет `LiteVoxLoader`).

## 2. Настройка `LiteVoxLoader`

1. Создайте пустой GameObject (например, `VoxelRoot`).
2. Добавьте компонент **LiteVoxLoader**.
3. В инспекторе задайте:
   - **Model Path** — путь **внутри** `StreamingAssets` без расширения.
     - Пример: если файл лежит в `Assets/StreamingAssets/Models/ship.vox`, то значение `Models/ship`.
   - **Model Index** — индекс модели внутри `.vox` (если файл содержит несколько моделей).
   - **Manager** — ссылка на `LiteVoxelManager` (можно оставить пустым, тогда используется `LiteVoxelManager.Instance`).

> После старта сцены `LiteVoxLoader` создаёт дочерний объект с `DynamicVoxelObj`, `MeshFilter`, `MeshRenderer` и `MeshCollider`, и загружает данные из `.vox` через `VoxFileDataProvider`.

## 3. Подключение `LiteLaserDestroy` и триггер

1. Добавьте **LiteLaserDestroy** на объект‑контроллер (например, на камеру или отдельный объект‑«оружие»).
2. В инспекторе:
   - **Ray Origin** — трансформ точки, откуда будет запускаться луч. Если не задан, используется `Camera.main`.
   - **Max Distance** — максимальная дальность луча.
3. Триггер:
   - ЛКМ (Mouse Button 0) — разрушение в точке попадания луча.

## 4. Использование `LiteExplosion`

1. Добавьте **LiteExplosion** на объект, в точке которого должна происходить «вспышка».
2. В инспекторе настройте:
   - **Explosion Radius** — радиус разрушения (пример: `6`).
3. Варианты запуска:
   - Автоматически при старте сцены (вызывается `Start()` → `Explode()`).
   - Вручную из кода: `GetComponent<LiteExplosion>().Explode();`

Ограниченный набор параметров:
- Только **радиус** (`Explosion Radius`).
- Нет силы/отбрасывания и каскадных эффектов — выполняется одноразовое сферическое разрушение.

## Мини‑чек‑лист запуска

- [ ] В сцене есть **LiteVoxelManager** с материалом и (опционально) `MeshSettings/DynSettings`.
- [ ] `.vox` файл лежит в `Assets/StreamingAssets/...`.
- [ ] `LiteVoxLoader` указывает корректный **Model Path** без расширения.
- [ ] На камере или объекте‑контроллере есть **LiteLaserDestroy** (если нужно «стрелять»).
- [ ] Для «взрыва» в сцене есть объект с **LiteExplosion**.

## Ожидаемое поведение

- При запуске сцены `LiteVoxLoader` создаёт воксельный объект и загружает модель из `.vox`.
- При удержании ЛКМ (`LiteLaserDestroy`) в месте попадания луча вырезается сферическая область.
- При запуске или ручном вызове `LiteExplosion.Explode()` в радиусе происходит одноразовое разрушение вокселей.
